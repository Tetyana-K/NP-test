HTTP (HyperText Transfer Protocol) — це протокол обміну даними між клієнтом і сервером.
Клієнт — це, наприклад, браузер або програма на C#.
Сервер — це вебсайт або API, який відповідає на запити.
В основі протоколу лежать запити, заголовки та коди результатів.

Основні методи:
	GET	Отримати дані із сервера
	POST	Надіслати дані на сервер
	PUT	Оновити ресурс
	DELETE	Видалити ресурс
Коди результатів:
	200	Успішно
	201	Створено
	204	Немає вмісту
	400	Помилка клієнта (Bad request)
	401	Неавторизовано (Unauthorized)
	403	Заборонено (Forbidden)
	404	Не знайдено (Not Found)
	500	Помилка сервера (Internal Server Error)

REST (Representational State Transfer) — це стиль побудови веб-сервісів,
де дані передаються через звичайні HTTP-запити.
API (Application Programming Interface) — це "інтерфейс", через який програми можуть спілкуватися між собою.

Тобто REST API — це спосіб, як одна програма (наприклад, клієнт)
може обмінюватися даними з сервером через HTTP.

Основна ідея REST

У REST є ресурси (наприклад, cars, users, orders),
і з ними працюють через HTTP-методи:

HTTP метод	Дія	Приклад
	GET	Отримати дані	/cars = список усіх авто
	POST	Додати новий ресурс	/cars = додати авто
	PUT	Оновити існуючий ресурс	/cars/5 = змінити авто з id=5
	DELETE	Видалити ресурс	/cars/5 = видалити авто з id=5

Формат даних
Дані зазвичай передаються у форматі JSON, наприклад:
{
  "brand": "BMW",
  "model": "X5",
  "year": 2020
}
